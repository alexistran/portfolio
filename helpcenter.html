Williams-Sonoma

Help Center


Help Center is a website accessible through the Room Planner app that gives designers and everyday consumers the ability to find answers to any questions they have about Room Planner on their own.




Spring 2019

Role: Lead Product Designer

Tools: Sketch, Invision, Adobe Illustrator, Draw.io, Google Docs





OVERVIEW

What's the Problem?

There are two versions of Room Planner— one for your everyday at-home consumer, and an enterprise version for Williams Sonoma's professional interior designers. If a user needs help with anything related to Room Planner, they have to email customer support. This is not the most quickest or efficient way to get help, especially if they just have a simple question.

There was no way for users to get help more self-sufficiently and fast.

Goal:

By creating an easy-to-use and straightforward help center that doesn't make getting help a drag, we believe that users will be able to get the help that they need quicker, and get a better understanding of all that Room Planner has to offer.

Contributions:

As the lead designer, I:
  • Conducted UX discovery phase (Market Research, Feature Prioritization)
  • Designed site map, user flows, wireframes and prototypes
  • Led usability testing sessions
  • Created digital illustrations for the website
  • Wrote UI documentation for engineers

THE SOLUTION

Making a tedious task more enjoyable


Design Decisions:
Sometimes finding help on your own is overwhelming. I kept the home screen of the help center very simple, really only allowing users to choose between 3 options: FAQ's, Updates, and a Contact form if they need more help. I created illustrations for each category to not only illustrate what each category is, but to bring a little more joy onto the page. There are also several access points for the user to get back to Room Planner after getting the help that they need.



Retaining familiarity


Design Decisions:
I retained the same menu bar style that exists in Room Planner so that it's very clear to the user the connection between the application and the help center. The menu bar allows access to all the content in the help center that is on the front page, from any page.



Structuring the help


Design Decisions:
The content in the FAQ's and Updates are structured such that all of the higher level categories are easy to access on the stickied left side panel. If the user still can't find what they're looking for from the FAQ's, Updates, and from searching, there is an easy to access contact button on every page.



The last resort


Design Decisions:
If a user can't get the answers that they need on the help center, there is still hope. Depending on if they're using the consumer version or pro version of Room Planner, there is a specific process for getting help. The contact modal pops up and grays out everything around it so that the user can focus on the task at hand.




TAKEAWAYS

What I Learned

  • Sometimes you can end up back on a previous concept, and that’s frustrating, but it's just part of the process.
  • People don't really think of help centers as “delightful experiences”, but that won’t stop me from trying.
  • Designing for two different users doesn't necessarily mean the experiences need to be significantly different.
  • Design with edge cases in mind! (what'll happen if a user types this, what if the user is on the wrong site, etc)
  • Copy is so important!
  • Hierarchy is so important in making dense information more consumable. This can be achieved in a variety of ways, like playing with text, color, spacing, etc.
