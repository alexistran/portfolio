Williams-Sonoma

Capture Mode


Room Planner is an application integrated across all Williams-Sonoma brands (West Elm, Pottery Barn, Williams Sonoma Home) that allows users to plan out room layouts and visualize Williams Sonoma products within that space. Capture Mode is a new feature for Room Planner that helps in-house designers better understand and design for their clients’ space, as well as capture eye-level perspective photos that sell.




Winter 2019

Role: Product Designer

Team: Product Manager, 2 R&D Engineers, 1 BE Developer, 1 FE Developer, 2 Designers (including myself)

Tools: Sketch, Invision, Framer, Draw.io, Google Docs





OVERVIEW

What's the Problem?

Our R&D team has developed the ability to generate images from a room plan at an eye-level perspective. This is significant because the only perspective the Room Planner currently displays is from a top view. This new perspective captures furniture detail that cannot be seen from a top view.

How might we utilize this technology to help designers at Williams Sonoma better design their clients’ ideal spaces?

Constraints:

  • This feature will only be available in the pro version of the app (only for Williams Sonoma in-house interior designers).
  • It currently takes at most 1 second to generate 1 image.
  • It's easier for the engineers to implement specific furniture pieces as the focal point of the image because it has set coordinates, rather than have the user arbitrarily click anywhere.


Goal:

We want to leverage the new technology in a way that is really easy to use (a lot of the designers are older folks who are not as tech-savvy) and is a purposeful tool for the designer so that they can best create for their clients, thus driving sales.

Contributions:

As a designer, I:
  • Helped conduct market research and ideation sessions
  • Designed user flows, wireframes and prototypes for capture mode (my co-designer worked on the photo gallery)
  • Led usability testing sessions
  • Wrote UI documentation for engineers





DISCOVERY

Market Research

We conducted research on other relevant online experiences that allow users to view their surroundings in perspective; there was no exclusive focus on room planning tools specifically. We wanted to understand how other tools utilized different perspective views and what made them successful or unsuccessful. Quick pros and cons were captured from each tool.



Google Maps Streetview
  • (+) The 360 views are from the perspective of the person, so it feels very life-like and realistic
  • (+) There are transparent arrows on how to move to the next view
  • (+) The transition of moving to another view is a zoom-like action that makes you feel like you’re walking fast. It's a very natural transition.


Room Sketcher
  • (+) Clear entry point to access gallery photos
  • (-) Doesn't let you view snapshot before you take it
  • (-) Not clear how to save photos


Living Spaces
  • (+) Renders in real time, viewable on top right corner
  • (-) UI is clunky and oversaturated with features
  • (-) Really long load times



Wayfair
  • (+) Allows users to view their room in any angle they want
  • (-) Doesn't allow users to take snapshots of their room, can only save the whole room
  • (-) Saving the room exits out of the room planner entirely

Based off of our constraints, and how we understood the tech that was developed, we decided to explore the idea of image capturing. We initially moved towards the direction of having a live view feature that would aid users as they designed, but because we cannot render images fast enough, this concept was not doable for our timeline.



User Flow

I created a user flow to understand the different paths that the user could take using this new feature to understand how it would tie to the existing Room Planner app functionality and navigation, and catch any edge cases we didn't consider yet. This flow has gone through several rounds of iterations.







DESIGN

Ideation

Utilizing the crazy 8's method, I quickly came up with 8 different versions of the main screens: capture mode entry point, item selection, and photo capture modal. With the help of my co-designer, we selected 5 of the 8 ideas to explore.






User Testing + Feedback

Several variations of the key interaction points, and iterations on the variations, were created and tested. We decided to test using wireframes because we wanted to quickly get feedback, and at this point we were looking for feedback on layout and functionality, not feedback on the visuals. We had users walk and talk through how they would use this new perspective feature. Here are a few of the screens we created:





Key Pieces of Feedback:
  • Distinguish the entry point more, this feature is similar to a new mode, it shouldn't blend into the existing modes.
  • Item selection was too similar to an existing interaction that it confused people.
  • Once in capture mode, make it clear what the user is supposed to do. Provide some sort of instructions.




THE SOLUTION

Entering Capture Mode


Design Decisions:
Capture Mode is an entirely new mode, on the same level as Build and Furnish, in the sense that they alter what you are able to do on the canvas. I made it stand out as a feature using color and iconography. The round button visually balances the page with the round Light tool. Hovering over the button informs users what this feature is.



Selecting a focal point


Design Decisions:
Due to an engineering request to have users select an item to render an image, I created this mediary screen to allow users to select an item before the photo capture modal can appear. Following the principle of progressive disclosure, this screen presents only what's related to the task at hand— selecting a focal point.



Loading the Render


Design Decisions:
One of our constraints was that it would take at most 1 second to generate a render. Each focal point has 3 different renders to capture the different angle views, thus it would take at around 3 seconds to load. With the expectation that many users would be using this, we prepared for the worst case scenario, by creating a loading screen that would inform users as they waited.



Taking photos in the Photo Capture Modal


Design Decisions:
The goal of this modal was to be easy to use, and not overwhelm the user. The key areas of interaction are changing the photo angles, capturing the image, and viewing the photo gallery. There is a photo count to inform the user how many more photos they can capture. We added a limit to prevent users from taking up more resources than we are able to allocate.



Viewing your photos





Design Decisions:
With the photos that the user has taken, they are able to view and create collages with them. This helps to understand the design of the space as a whole, and to share with customers to ensure every detail meets their expectations. To allow for customization, designers are able to select different layout styles for their collages, and save/share them as PDF's.




TAKEAWAYS

What I Learned

  • Inspiration can be found anywhere, even in very different products.
  • Be prepared for the worst case scenario, especially in regards to engineering/resource limitations. We had an idea involving a live render, but since it takes so long to render an image currently, that wasn't possible and we had to be flexible and pivot ideas quickly.
  • It doesn't hurt to inform the user what they are going to do with more text sometimes. The most minimalistic design might not be the clearest design.
